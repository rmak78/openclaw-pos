# Accessibility Baseline

## Target
- WCAG 2.2 Level AA for core POS workflows

## Contrast
- Normal text: 4.5:1
- Large text: 3:1
- Focus indicators/components: 3:1

## Keyboard
- Full keyboard operability
- Visible focus indicators
- Modal focus trap + focus return

## Forms
- Programmatic labels
- Text-based error messaging and associations
- Clearly marked required fields

## Screen Readers
- Semantic landmarks and table header scopes
- Accessible names for icon buttons
- Live region announcements for state changes where needed

## Practical QA Checklist
- Tab-only checkout journey
- Screen reader smoke test (NVDA/VoiceOver)
- EN/UR/AR clipping and expansion checks
