# Onboarding Checklist: New Country / Region / Branch

Use this checklist when launching into a new market, region, or branch cluster.

## 1) Market and Regulatory Setup (Country)

- [ ] Currency, decimal rules, and rounding policy confirmed.
- [ ] Tax model configured (VAT/GST/sales tax rules).
- [ ] Legal invoice/receipt requirements documented.
- [ ] Data retention and privacy obligations confirmed.
- [ ] Required compliance approvals captured.

## 2) Operating Model Setup (Region)

- [ ] Region calendar, holidays, and business week configured.
- [ ] Time zone and daylight-saving behavior validated.
- [ ] Regional price lists and promotion policies defined.
- [ ] Language pack target(s) identified and translation owner assigned.
- [ ] Support and escalation contacts assigned.

## 3) Branch Master Data Readiness

- [ ] Branch profile created (name, address, geo, hierarchy).
- [ ] Till count and device IDs registered.
- [ ] Store hours and shift templates configured.
- [ ] Tender types enabled per branch policy.
- [ ] Inventory source mapping validated.

## 4) Access and Roles

- [ ] Role matrix approved (Cashier/Supervisor/Manager/Support).
- [ ] User provisioning completed with least-privilege access.
- [ ] MFA/security policy applied (if required).
- [ ] Supervisor authorization thresholds configured.

## 5) Hardware and Connectivity

- [ ] Till hardware installed and labeled.
- [ ] Printer, scanner, and payment terminal tests passed.
- [ ] Network quality baseline recorded.
- [ ] Offline fallback policy reviewed with branch team.

## 6) Operational SOP Readiness

- [ ] Open shift SOP trained and signed off.
- [ ] Close shift SOP trained and signed off.
- [ ] Offline mode SOP trained and signed off.
- [ ] Sync recovery path and support contacts shared.
- [ ] Incident log process introduced.

## 7) Reporting and Finance Controls

- [ ] Daily branch sales report validated.
- [ ] Shift close reconciliation report validated.
- [ ] Settlement and payout report validated.
- [ ] Variance handling workflow approved by finance.

## 8) Pilot and Go-Live

- [ ] Pilot branch selected and staffed.
- [ ] UAT scenarios completed (online + offline + recovery).
- [ ] Hypercare window scheduled (first 1-2 weeks).
- [ ] Release notes and communication pack sent.
- [ ] Go/No-Go decision recorded with approvers.

## 9) Post-Go-Live (First 30 Days)

- [ ] Daily KPI review cadence active.
- [ ] Offline incidents tracked and trended.
- [ ] Sync conflict dashboard monitored.
- [ ] Training gaps captured and remediated.
- [ ] Retrospective completed and actions assigned.
