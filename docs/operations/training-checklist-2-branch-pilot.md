# Training Checklist — 2-Branch Pilot

**Document ID:** OPS-TRAINING-PILOT-2B-1.0  
**Version:** 1.0 (Pilot Readiness)  
**Audience:** Branch Managers, Shift Supervisors, Cashiers, Support  
**Branches:** Pilot Branch A, Pilot Branch B  
**Owner:** Training Lead + Product Ops

---

## 1) Training Objective
Ensure both pilot branches can operate safely and independently on core workflows:
- Open Shift
- Close Shift
- Offline Operations
- Sync Recovery awareness and escalation

---

## 2) Pilot Cohorts & Target Completion

| Cohort | Branch A | Branch B | Target Completion | Minimum Pass |
|---|---:|---:|---|---|
| Branch Manager | 1 | 1 | T-5 days | 100% |
| Shift Supervisors | 2 | 2 | T-4 days | 100% |
| Cashiers | 6 | 6 | T-3 days | 90% |
| Support L1 Desk | Shared | Shared | T-3 days | 100% |

---

## 3) Training Delivery Plan

### Format
- Instructor-led walkthrough (2 hours)
- Hands-on sandbox simulation (2 hours)
- Branch floor drill (1 hour)
- Assessment + sign-off (45 minutes)

### Materials Required
- SOP packet v0.1 print/digital
- Scenario cards (normal, outage, mismatch)
- Till simulator credentials
- Incident log and reconciliation templates

---

## 4) Module Checklist (By Role)

## Module A — SOP Foundations (All Roles)
- [ ] Explain pilot goals, risks, and escalation path
- [ ] Understand role boundaries and approvals
- [ ] Locate SOP packet and quick-reference sheets
- [ ] Confirm comprehension via short quiz

## Module B — Open Shift Drill
- [ ] Login with individual credentials
- [ ] Perform float count + dual verification
- [ ] Execute readiness checks (printer/scanner/terminal/sync)
- [ ] Produce opening proof and set till Ready
- [ ] Complete open-shift checklist without assistance

## Module C — Close Shift Drill
- [ ] Run close shift summary
- [ ] Count and reconcile cash
- [ ] Record variance reason code correctly
- [ ] Supervisor approval workflow
- [ ] Generate end-of-shift packet and handover log

## Module D — Offline Mode Drill
- [ ] Identify offline trigger signal
- [ ] Activate offline workflow and incident start log
- [ ] Process allowed transaction types only
- [ ] Explain restricted transaction types to customer
- [ ] Verify offline queue indicator and local receipt behavior

## Module E — Sync Recovery & Escalation Drill
- [ ] Recognize reconnect + sync status transitions
- [ ] Validate queue drains after recovery
- [ ] Identify conflict classes and who owns each
- [ ] Raise escalation ticket with required fields
- [ ] Confirm closure criteria and manager sign-off path

## Module F — Compliance, Tax & Audit Hygiene
- [ ] No shared accounts / no PIN sharing
- [ ] Correct use of adjustment and variance reason codes
- [ ] Proper handling of voids/refunds approvals
- [ ] Evidence and document retention basics

---

## 5) Assessment & Certification

### Knowledge Check (Written/Oral)
- [ ] Minimum score threshold achieved
- [ ] Missed topics remediated same day

### Practical Assessment (Observed)
- [ ] Open shift completed correctly (time + accuracy)
- [ ] Close shift completed correctly (reconciliation + docs)
- [ ] Offline scenario handled per SOP
- [ ] Escalation raised with complete incident details

### Certification Outcome
- [ ] PASS — cleared for pilot day operations
- [ ] CONDITIONAL — supervised shifts required
- [ ] FAIL — retraining required before go-live
